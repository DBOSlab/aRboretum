# All about aRboretum

## Introduction

`aRboretum` is an integrated R package designed to automate the
production of botanical collection labels, from taxonomic data
extraction to interactive web labels and print-ready QR codes. It can be
used for arboreta, botanic gardens, educational trails, and other living
plant collections.

The package offers:

- multilingual output in Portuguese, English, French, and Spanish;
- synonym resolution and integration of data from [Flora e Funga do
  Brasil (FFB)](https://reflora.jbrj.gov.br/reflora/listaBrasil/) and
  [Plants of the World Online (POWO)](https://powo.science.kew.org/);
- interactive HTML labels with species information, text-to-speech,
  distribution maps, photo slideshows, and optional personal audio
  recordings;
- a searchable minisite index linking all generated labels;
- printable QR-code labels in different layouts;
- one flexible additional language slot for community-based
  accessibility through `add_lang`.

All functions are designed to work together. This vignette provides a
technical overview of the package workflow, main arguments, expected
data structure, and common customisation options.

### Installation

``` r
# Install from GitHub
if (!require("devtools")) install.packages("devtools")
devtools::install_github("DBOSlab/aRboretum")
```

Main dependencies are installed automatically with the package.

## Internal data structures

#### The internal translation dictionary

The package contains an internal translation table used to build species
descriptions in the built-in languages. It includes terms and sentence
fragments related to:

- taxonomy;
- endemism and establishment means;
- phytogeographic domains and vegetation types;
- IUCN categories;
- genus diversity notes;
- sentence-building connectors.

The helper functions `.tr_dict()` and `.phrase_generator()` use this
dictionary to produce natural-language species summaries.

#### Botanical regions mapping

The package also relies on an internal mapping between botanical
division codes, countries, and continents. This allows distribution data
from [POWO](https://powo.science.kew.org/) to be converted into readable
geographic statements.

Not every column needs to be filled, but the core taxonomic and
distribution fields are required for full functionality.

## Main workflow

A typical `aRboretum` workflow consists of the following steps:

1.  extract and structure species data with
    [`arboretum_data()`](https://DBOSlab.github.io/aRboretum/reference/arboretum_data.md);
2.  optionally prepare folders for personal audio with
    [`arboretum_audios()`](https://DBOSlab.github.io/aRboretum/reference/arboretum_audios.md);
3.  optionally prepare folders for species photos with
    [`arboretum_photos()`](https://DBOSlab.github.io/aRboretum/reference/arboretum_photos.md);
4.  generate interactive HTML labels with
    [`arboretum_labels()`](https://DBOSlab.github.io/aRboretum/reference/arboretum_labels.md);
5.  create a searchable microsite with
    [`arboretum_minisite()`](https://DBOSlab.github.io/aRboretum/reference/arboretum_minisite.md);
6.  create printable QR-code labels with
    [`arboretum_qrcodes()`](https://DBOSlab.github.io/aRboretum/reference/arboretum_qrcodes.md).

## Function overview

## `arboretum_data()`

### What it does

[`arboretum_data()`](https://DBOSlab.github.io/aRboretum/reference/arboretum_data.md)
extracts and merges species information from Flora e Funga do Brasil and
Plants of the World Online. It resolves synonyms when possible, merges
overlapping information, and returns a standardised dataset that can be
used by the rest of the package.

The output also includes empty user-editable columns such as
`plant_uses_*` and `free_notes_*`, which can later be used to enrich
labels in the built-in languages.

### Example

``` r
spp_list <- c("Luetzelburgia bahiensis", "Paubrasilia echinata")

extracted_data <- arboretum_data(
  spp_list = spp_list,
  save = TRUE,
  format = "xlsx",
  dir = "arboretum_data"
)
```

### Notes

- Synonyms are resolved to accepted names when possible.
- Missing values from one source may still be complemented by the other.
- The saved output can be edited manually before generating labels.

## `arboretum_audios()`

### What it does

[`arboretum_audios()`](https://DBOSlab.github.io/aRboretum/reference/arboretum_audios.md)
prepares a folder structure for personal species audio recordings. For
each species and each selected language, the function creates one folder
where users can place audio files.

It also generates an HTML guide to facilitate the recording workflow.

### Example

``` r
arboretum_audios(
  data_path = "arboretum_data/arboretum_data.xlsx",
  printed_lang = c("pt", "en", "fr", "es"),
  add_lang = "PANARA",
  verbose = TRUE
)
```

### Notes

- Built-in languages are `"pt"`, `"en"`, `"fr"`, and `"es"`.
- The optional `add_lang` argument creates one additional language
  folder per species, such as `PANARA`.
- This is useful when preparing personal audio in a community or local
  language without translating the whole interface.

## `arboretum_photos()`

### What it does

[`arboretum_photos()`](https://DBOSlab.github.io/aRboretum/reference/arboretum_photos.md)
creates one photo folder per species so that users can add personal or
curated images to be embedded later in the HTML labels.

### Example

``` r
arboretum_photos(
  data_path = "arboretum_data/arboretum_data.xlsx",
  verbose = TRUE
)
```

### Notes

- Existing folders are preserved.
- Photos are optional and only appear in labels when files are found in
  the expected species folder.

## `arboretum_labels()`

### What it does

[`arboretum_labels()`](https://DBOSlab.github.io/aRboretum/reference/arboretum_labels.md)
generates one standalone HTML label per species. Each label may include:

- multilingual species text;
- taxonomic information;
- world and Brazil distribution maps;
- browser-based speech synthesis;
- personal audio recordings when available;
- photo slideshows when available;
- IUCN iconography;
- institutional and package logos.

### Example

``` r
arboretum_labels(
  data_path = "arboretum_data/arboretum_data.xlsx",
  audio_dir = "arboretum_audios",
  photo_dir = "arboretum_photos",
  printed_lang = c("pt", "en"),
  add_lang = "TUKANO",
  path_to_logo = "path/to/institution_logo.png",
  logo_url = "https://www.example.org",
  dir = "species_labels",
  verbose = TRUE
)
```

### The `add_lang` workflow

The `add_lang` argument provides a flexible, interim pathway for
including one additional community or local language in species labels
and associated audio, without requiring translation of the full website
interface.

When `add_lang` is supplied:

- [`arboretum_labels()`](https://DBOSlab.github.io/aRboretum/reference/arboretum_labels.md)
  looks for a column named `full_phrases_ADD_LANGUAGE`;
- if that column contains non-empty text, the extra language is added as
  an additional text option in the label;
- the custom language text is read directly from
  `full_phrases_ADD_LANGUAGE`;
- this extra language does **not** pass through the standard
  phrase-generation and translation workflow.

This is especially useful in collaborative work with Indigenous peoples
or other local communities, where species-level accessibility may be
more important than translating the full minisite interface.

### Notes

- Personal audio files take priority over browser speech synthesis.
- For custom languages added through `add_lang`, personal recordings are
  recommended because browser voices may not be available.
- Photos and maps are included only when available.

## `arboretum_minisite()`

### What it does

[`arboretum_minisite()`](https://DBOSlab.github.io/aRboretum/reference/arboretum_minisite.md)
builds a searchable index page linking all generated species labels. It
can also include simple collection summaries and charts when the species
dataset is provided.

### Example

``` r
arboretum_minisite(
  labels_dir = "species_labels",
  data_path = "arboretum_data/arboretum_data.xlsx",
  site_title = "Garden Collection",
  printed_lang = c("en", "pt", "es", "fr"),
  logo = "path/to/logo.png",
  logo_url = "https://www.example.org",
  group_by_family = TRUE
)
```

### Notes

- The interface language order follows the order of `printed_lang`.
- The minisite can work without `data_path`, but summary panels may be
  reduced.

## `arboretum_qrcodes()`

### What it does

[`arboretum_qrcodes()`](https://DBOSlab.github.io/aRboretum/reference/arboretum_qrcodes.md)
creates printable QR-code labels that can link to species HTML labels,
external URLs, or source records. The output can be generated in
minimalist or complete layouts.

### Example

``` r
arboretum_qrcodes(
  data_path = "arboretum_data/arboretum_data.xlsx",
  base_url = "https://myinstitution.org/collection",
  layout = "minimalist",
  dir = "qrcodes_mini"
)

arboretum_qrcodes(
  data_path = "arboretum_data/arboretum_data.xlsx",
  base_url = "https://myinstitution.org/collection",
  layout = "complete",
  width = 6.0,
  length = 8.0,
  id_code = "catalogue_number",
  printed_lang = "en",
  path_to_logo = "path/to/logo.png",
  color = "#1a2e1a",
  format = "pdf",
  dir = "qrcodes_complete"
)
```

### Notes

- When no custom URL is provided, the function falls back to source URLs
  when possible.
- Output can span multiple pages automatically.

## Practical example with one additional language

The example below illustrates a simple workflow where Portuguese and
English remain the main interface languages, while Tukano is added as a
community language for species labels and optional personal recordings.

``` r
arboretum_audios(
  data_path = "arboretum_data/arboretum_data.xlsx",
  printed_lang = c("pt", "en"),
  add_lang = "TUKANO"
)

arboretum_labels(
  data_path = "arboretum_data/arboretum_data.xlsx",
  audio_dir = "arboretum_audios",
  printed_lang = c("pt", "en"),
  add_lang = "TUKANO",
  dir = "species_labels"
)
```

For this workflow, the input file should contain a column named
`full_phrases_ADD_LANGUAGE` with the full text to display in Panará for
each species.

## Defensive checks and internal helpers

Several internal helper functions provide validation and error
prevention.

### `.arg_check_spp_list()`

- trims whitespace;
- standardises input species names;
- checks for invalid entries.

### `.arg_check_dir()`

- validates output directory names;
- removes trailing slashes when needed.

### `.arg_check_printed_lang()`

- standardises language codes to lower case;
- removes duplicates;
- validates built-in language codes.

### `.read_species_data()`

- checks file existence and extension;
- reads CSV or Excel input;
- validates required columns.

### `.copy_folder_progress()`

- copies user-provided audio or photo folders into output directories;
- shows progress feedback when many files are involved.

### Phrase-generation helpers

Helpers such as `.phrase_generator()`, `.tr_dict()`,
`.botdiv_to_countries()`, and `.add_genus_curiosity_notes()` support the
construction of species descriptions in the built-in languages.

## Conclusion

`aRboretum` provides a practical pipeline for creating modern botanical
collection labels, from taxonomic data retrieval to multilingual HTML
pages and printable QR codes. The package combines automation with
flexibility, allowing users to enrich labels with notes, uses, photos,
personal audio, and one additional community-based language through
`add_lang`.
